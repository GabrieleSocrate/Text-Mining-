# Text-Mining- Financial News Sentiment Analysis

## Project Overview
The goal of this project is to perform sentiment analysis on real financial news from 2018 to 2023, scraped from Yahoo Finance. Several neural network models were implemented to classify each news in one of the three plausible labels: "Positive", "Negative" or "Neutral".

# BiLSTM Model

## Overview
This section describes the BiLSTM-based sentiment analysis component of the project.

The BiLSTM model is designed to classify the sentiment of financial news articles into three categories: **positive**, **negative**, and **neutral**.  
It is trained on the project dataset after preprocessing, balancing, tokenization, and sequence padding. The model uses **pre-trained GloVe embeddings** to improve the representation of textual information and to capture semantic relationships between words.

In addition to training and evaluation, this component generates all the artifacts required to reuse the model without retraining and to integrate its outputs into the dashboard.

---

## Main Functionality
The BiLSTM pipeline includes the following steps:

1. Loading and preprocessing the dataset  
2. Balancing the sentiment classes  
3. Tokenizing and padding the input texts  
4. Loading pre-trained GloVe embeddings  
5. Performing hyperparameter tuning  
6. Training the final BiLSTM model  
7. Evaluating the model on the test set  
8. Exporting the results and weights required by the dashboard  

---

## Files Generated by the BiLSTM Component

After execution, the BiLSTM pipeline generates the following files and folders:

### `bilstm_glove_checkpoints/`
This folder contains the files required to reconstruct the BiLSTM model without retraining.  
It includes the tokenizer, configuration files, and the exported model weights used by the dashboard.

**Location:** project root directory

---

### `bilstm_glove_gridsearch/`
This folder contains the results of the hyperparameter search and the final trained BiLSTM model.

It is required to preserve the training outputs and to allow the model to be reused without repeating the full optimization and training process.

**Location:** project root directory

---

### `bilstm_results.json`
This file stores the evaluation outputs of the BiLSTM model, including:
- confusion matrix
- classification report
- true labels
- predicted probabilities

These data are required by the dashboard to display the BiLSTM performance metrics and related visualizations.

**Location:** project root directory

---

## Dashboard Integration
The dashboard uses the files generated by the BiLSTM component to display:
- model performance metrics
- confusion matrix
- classification results
- comparative visualizations

If the required files are already available in their correct locations, the BiLSTM model can be used directly inside the dashboard **without retraining**.

---

## Notes
- The BiLSTM model uses **GloVe 100-dimensional embeddings**.
- The model was trained on a balanced dataset with three sentiment classes.
- If these files are missing, the dashboard will not be able to load the BiLSTM model or display its metrics correctly.
- If all files are already available, it is not necessary to retrain the model from scratch.

  

### 3. Bert Models
#### DistilBert fine-tuning: 
- transformers model which was pretrained by knowledge distillation using BERT as foundational model.

#### FinBert feature extraction: 
- pre-trained model to analyze sentiment of financial text, built by further training the BERT language model in the finance domain.

#### Bert base fine-tuning:
- transformers model pretrained on generic data, considered a foundational model for sequence classification, token classification or question answering tasks.

### 4. RAG

- loading and splitting the dataset into small text chunks.

- creating embeddings and storing them in a FAISS index for efficient similarity-based retrieval of relevant context.

- implementing a question-answering system that retrieves top-k relevant chunks, builds an augmented prompt, and asks a local LLM (via Ollama) to generate context-grounded answers.

# Dashboard Overview and Setup

## Introduction
The dashboard provides an interactive environment to explore and analyze the results generated by the project. It includes:

- Exploratory Data Analysis (EDA)
- Model performance metrics
- A BiLSTM model
- Three BERT-based models
- A Retrieval-Augmented Generation (RAG) component
- A dedicated page for company-level sentiment analysis and visualization

The dashboard relies on precomputed data, model outputs, and weights.  
To use it correctly, all project files must be executed beforehand in order to generate the required resources.

---

## Important Note
Although the dashboard is contained within the `dashboard/` folder, **not all the files it depends on are located inside this directory**.

Several required resources (e.g., model weights, results, FAISS index, dataset) are stored in the **root directory** of the project.  
Make sure the entire project has been executed and all files are correctly placed before running the dashboard.

---

## Running the Dashboard

To launch the dashboard, run the following command from the project root directory:

```bash
python dashboard/app/app.py
```
